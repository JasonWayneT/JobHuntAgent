## 1. Executive Summary
JobAgent is an end-to-end local application and automation platform tailored for the modern Product Management job search. It bridges the gap between raw job postings and premium, perfectly tailored application materials (Resumes, Cover Letters, Interview Cheat Sheets), while maintaining 100% grounding in verified work experience. 

With v4.0, JobAgent transitions from a collection of CLI scripts into a **Full Local Command Center**, featuring a live data-driven GUI dashboard, real-time application state tracking, and centralized LLM utility structures. It completely replaces external tracking tools like Google Sheets.

## 2. Target Audience & Positioning
- **Primary Persona:** Jason Taylor.
- **Target Role:** Mid-level Product Manager (3-6 years experience).
- **Specialization:** B2B SaaS, Platform Infrastructure, Data Integrity, and Platform Stabilization.

## 3. Product Vision & Core Pillars
1. **Automate the Grind:** Eliminate manual job searching and iterative resume tailoring via local LLM orchestration.
2. **Absolute Ground Truth:** Use a rigid "Ground Truth" data model (`workExperience.md`) to prevent AI-generated seniority inflation, invalid tool claims, or "hallucinated" leadership.
3. **Strategic Fuel:** Provide deep corporate intelligence for every role to enable high-intent, "prepared-for-day-one" applications using dual-agent research (Perplexity + Gemini).
4. **State-Aware Pipeline:** Centralize all tracking locally. Track every job from discovery to offer, including dates applied, rejection reasoning, and interview notes, via a unified browser dashboard.

## 4. Technical Architecture & Component Logic

The system is built entirely in Python, utilizing local filesystem storage and a lightweight local HTTP API server to interact with the frontend.

### 4.1 State Management (The Database)
State is managed centrally in `data/job_database.json`. The schema has been normalized to track deep temporal data:
- `job_key` (String): Unique identifier (e.g., "Company - Title").
- `company` & `title` & `url` (Strings): Core job identifiers.
- `status` (Enum): `Scouted` -> `Drafted` -> `Applied` -> `Interviewing` -> `Offered` | `Rejected` | `Archived`.
- `score` (Integer): 0-100 fit score generated by the Gatekeeper.
- `folder_path` (String): Relative path to generated assets (e.g., `submissions/company_name`).
- `date_scouted`, `date_applied`, `date_rejected` (Date Strings): Temporal tracking.
- `rejection_reason` (String): Extracted rationale for rejected roles.
- `notes` (String): Free-form user notes replacing Google Sheets comments.
- `source` (String): Auto-inferred extraction source (LinkedIn, Ashby, Greenhouse, etc.).

### 4.2 The GUI Command Center (`Job_Navigator.html` & `server.py`)
- **Backend:** A lightweight Python HTTP server (`server.py`) serving static assets and exposing a `PATCH /api/jobs/<id>` REST endpoint to write updates directly to the JSON database.
- **Frontend Dashboard:** A vanilla HTML/JS Single Page Application featuring a premium Glassmorphism aesthetic.
- **Features:** 
  - Dynamic pipeline stats bar counting active loads.
  - Interactive job cards grouped by pipeline stage.
  - Real-time search and filter functionality.
  - **Edit Modal:** Click any job card to dynamically update statuses, append notes, and log Application/Rejection dates (with auto-stamping).

### 4.3 Scouting Engine (The Hunter)
- **Source:** LinkedIn Jobs API (via browser subagent `/scout` workflow).
- **Filters:** Mid-Senior level, Remote/San Diego, Posted within 7 days.
- **Logic:** Matches job IDs and titles against `data/scouted_jobs_log.txt` to prevent duplicate pipeline ingestion. New roles are saved as `.txt` files in `/jobs/`.

### 4.4 Job-Fit Decision Engine (The Gatekeeper)
Defined in `.agent/rules/job_fit_engine.md`. Evaluates the `/jobs/` queue.
- **Stage A: The Fast Gate (Instant Disqualifiers):**
  - **Blocklist Titles:** Senior, Staff, VP, Head, Principal, Lead, Director, *Growth*.
  - **Solo PM Trap:** Rejecting "Founding PM" or roles in small startups (<50 people).
  - **Management Mismatch:** Rejecting roles requiring direct people management of PMs.
  - **Industry Blocklist:** Gambling, Sports Betting, Gaming, Ad Tech, Crypto, Web3.
- **Stage B: Full Scoring (100-Point Rubric):**
  - **Direct Leadership Match (25 pts):** Mentor-rich environments.
  - **Seniority Fit (25 pts):** IC optimization.
  - **Technical Depth (25 pts):** Integrations, data pipelines, internal platforms.
  - **Growth Potential (25 pts):** High-impact operational scaling.
- **The "Two-Anchor" Rule:** A **YES** requires a score &gt;= 78 and at least two overlaps with Jason's core pillars (Platform Stability, Data Migration, Security, B2B Scaling).

### 4.5 Intelligence Layer (The Research Engine)
- **Dual-Model Logic:** Uses **Perplexity (`sonar-pro`)** for live web-intel with automatic rate-limit fallback to **Gemini 2.5 Pro** with Google Search tools.
- **Output:** A structured JSON "Research Packet" categorized strictly by the `Research_Packet_Contract.md` (Company DNA, Competitors, Financials, Culture Reality Check).

### 4.6 Drafting Layer (The Bridge Engine)
- **Bridge Logic:** Actively translates "Platform/Infrastructure" history into outcomes matching the specific JD's needs. 
- **Claim Verifier V2:** A cynical auditor LLM prompt runs across all generated text. It is designed to aggressively rewrite sentences that contain hallucinated tools (e.g., Snowflake), inflated seniority (e.g., "Led a team"), or AI-fingerprints (em-dashes, "Furthermore", "Innovative").

### 4.7 Interview Prep Engine (The Coach)
- **Functionality:** Generates an `Interview_Cheat_Sheet.md`.
- **Structure:** 60-second problem-space pitch, "Why this company?" alignment, bridging past Cision stories (Bellwether, CPRE) to JD challenges, and highly technical reverse-interview questions.

## 5. Workflows & Orchestration

### 5.1 `/scout` Workflow
*Triggered via explicit agent command.*
1. Invokes Browser Subagent to parse LinkedIn.
2. Cross-references logs.
3. Downloads 3-4 high-intent `.txt` Job Descriptions into `/jobs/`.

### 5.2 `/evaluate` Workflow
*The primary action loop.*
1. Reads `jobs/` queue.
2. Applies Job-Fit Engine Gatekeeper rules.
3. Automatically creates `/submissions/[Company]` architecture for winners (>= 78).
4. Generates all application assets (Resume, Cover Letter).
5. Invokes PDF conversion engines.
6. Invokes Research and Cheat Sheet engines.
7. Commits outcome back to `data/job_database.json` and transitions state to `Drafted`.

### 5.3 Shared Utility Layer (`scripts/utils.py`)
All downstream orchestrator scripts leverage a centralized `utils.py` module to ensure:
- Normalized relative pathing (`PROJECT_ROOT`-aware).
- Centralized `google.genai` initialization tracking `DEFAULT_MODEL` (`gemini-2.5-flash`).
- Built-in API rate-limit detection and exponential back-off retries (`call_llm()`).
